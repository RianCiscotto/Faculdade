1) Sobre o conceito de Backend, assinale a alternativa correta:
B) Backend aplica regras de negócio, acessa banco de dados e retorna respostas, geralmente em
JSON.

2) Em uma API REST, o princípio “Stateless” significa que:
C) Cada requisição deve conter todas as informações necessárias para ser processada.

3) Sobre o método GET, é correto afirmar que:
C) É considerado seguro (safe) e idempotente.

4) Qual situação representa corretamente o uso do método PUT?
C) Substituir completamente os dados de um recurso existente.

5) Um cliente envia a seguinte requisição: GET /tarefas?status=ABERTA
O comportamento correto da API deve ser:
C) Filtrar e retornar apenas tarefas com status ABERTA.

6) Um usuário tenta cadastrar um novo registro com um e-mail que já existe no banco. O
status code mais adequado é:
C) 409 Conflict

7) Um cliente autenticado tenta acessar um recurso para o qual não possui permissão. O
status correto é:
B) 403 Forbidden

8) Sobre idempotência, analise as afirmações:
I – GET é idempotente.
II – PUT é idempotente.
III – DELETE é idempotente.
IV – POST é idempotente.
Assinale a alternativa correta:
B) Apenas I, II e III
